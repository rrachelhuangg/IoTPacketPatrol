Project Architecture
The data used for this project is a 733,705 element dataset curated by Dyuti_Dasmahaptra on Kaggle and can be found here. The dataset was selected for its quality and large size, containing network traffic flows collected from Internet of Things (IoT) devices that were operating in environments affected by botnet attacks. Each row in the dataset represents a single network flow with packet sequence ID, used protocol, source and destination IP addresses, source and destination ports, and a multitude of defining network characteristics. Each network is given a label to identify if it is an attack, and is also subcategorized into its corresponding attack types, if applicable.
Univariate plots were created for the dataset's selected features. Due to the size of the dataset, these plots were created with only ~35% of the dataset, but display patterns applicable to the entire dataset. The majority of the networks catalogued were attacks, and the imbalanced dataset was addressed during model training with sci-kit learn's balanced class weight parameters. Deviations from typical mean packet inter-arrival time helps distinguish network traffic, large numbers of inbound connections may indicate a scanning or DDoS network attack.
The dataset was used to train a logistical regression model that classifies a flow as benign or malicious, and the specific categories and methods of the attack if applicable.
The dataset and trained model were stored in MongoDB's Atlas Database.
